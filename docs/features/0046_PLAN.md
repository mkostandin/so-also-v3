### Technical plan

1) Single scroll-owner architecture for app content
- Change the app shell to stop scrolling at the wrapper level and let routed views (List/Map/Calendar) own their own scroll containers.
- Edit `ui/src/App.tsx` `BottomTabsWrapper`: replace `overflow-y-auto` with `overflow-hidden` on the `flex-1` container that wraps `<Outlet />`. Keep `pb-16` for bottom tabs spacing.
  - Outcome: prevents nested scrolling with the inner view container, allowing the List view’s `overflow-y-auto` to work on mobile.

2) Harden List view scroll container for mobile
- Keep `div.flex-1.min-h-0.overflow-y-auto` in `ui/src/routes/ListView.tsx` as the single scroll container for content under the filters.
- Add a mobile momentum and touch-panning utility class to this container:
  - In `ui/src/index.css`, add a utility (no code here) to apply `-webkit-overflow-scrolling: touch;` and `touch-action: pan-y;`.
  - Apply that class to the List view’s scroll container alongside `overflow-y-auto`.
- Ensure parents already have `min-h-0` (they do in ListView and MapIndex) to prevent height collapsing that blocks scrolling.

3) Make dropdown list scroll reliably on mobile
- Increase menu height to viewport-relative and ensure touch-panning:
  - Update `ui/src/components/CommitteeFilter.tsx` menu container classes: use `max-h-[60vh]` (or similar) instead of the fixed `max-h-60`, and add the same mobile scroll utility (`-webkit-overflow-scrolling: touch;` and `touch-action: pan-y;`). Keep `overflow-y-auto`.
- Optional but recommended: render the dropdown menu in a portal to `document.body` to avoid clipping and nested scroll capture in complex layouts. Either:
  - Switch the custom dropdown to a Radix `Popover`/`Dialog` pattern which portals by default, or
  - Wrap the existing menu with `createPortal` to `document.body` while preserving positioning.

4) Re-scope aggressive global selection rule
- The global `* { user-select: none; }` can make some mobile browsers treat scroll gestures differently in nested overflow contexts.
- Narrow this rule to specific interactive elements only (e.g., buttons, tab triggers) and allow selection/normal behavior for divs/scrollable containers. Alternatively, explicitly set `user-select: auto;` on the List scroll container and the dropdown menu container via a utility class.

### Implementation updates (finalized)

The following changes were implemented to resolve mobile scrolling, align layouts, and ensure consistent behavior across views.

- App shell and scroll ownership
  - Kept `ui/src/App.tsx` simple: wrapper `min-h-screen flex flex-col`, content `flex-1 min-h-0` (no scrolling here).
  - Made `MapIndex` the sole scroll owner.

- Tabs header (consistent across views)
  - In `ui/src/routes/MapIndex.tsx`, tabs container is sticky, not fixed:
    - `sticky top-0 z-50 border-b bg-white/90 backdrop-blur dark:bg-gray-900/60`.
  - Content scroller (the element that scrolls all views) now has:
    - `overflow-y-auto pb-16 scroll-touch scroll-pan-y scrollbar-stable overscroll-none select-auto`.
    - `scrollbar-stable` prevents horizontal “slide” when the vertical scrollbar appears.
    - `overscroll-none` prevents pull-to-refresh and overscroll chaining on Chrome/Edge.

- Global touch/scroll CSS utilities added in `ui/src/index.css`
  - `.scroll-touch` → `-webkit-overflow-scrolling: touch; touch-action: pan-y;`
  - `.scroll-pan-y, .scroll-pan-y *` → `touch-action: pan-y !important;`
  - `.scroll-pan-x, .scroll-pan-x *` → `touch-action: pan-x !important;` (for horizontal scrollers)
  - `.scrollbar-stable` → `scrollbar-gutter: stable both-edges;`
  - `.overscroll-none` → `overscroll-behavior: none;`
  - Disabled browser pull-to-refresh globally: `html, body { overscroll-behavior-y: none; }`.

- Event Type filter (horizontal pan on touch)
  - `ui/src/components/EventTypeFilter.tsx` container now includes `scroll-pan-x` plus existing `overflow-x-auto` to allow horizontal panning while the page scrolls vertically.

- Committee filter dropdown
  - `ui/src/components/CommitteeFilter.tsx` dropdown menu uses `max-h-[60vh] overflow-y-auto scroll-touch scroll-pan-y` for reliable mobile scrolling without affecting page scroll.

- List view layout (consistent with Map/Calendar)
  - `ui/src/routes/ListView.tsx`
    - Filters are `sticky top-0 z-40 bg-white dark:bg-gray-900 border-b` (no extra paddings), matching other views.
    - List content remains `flex-1 min-h-0 overflow-y-auto`, so the list scrolls beneath the filters and reaches the “Load More” button.

- Calendar view filters
  - `ui/src/routes/CalendarView.tsx` uses the same `sticky top-0 z-40 bg-white dark:bg-gray-900 border-b` pattern for the filters to align with List and Map.

- Debug tooling
  - A temporary `TouchScrollDebugger` overlay was used during diagnosis and removed from the app runtime.
